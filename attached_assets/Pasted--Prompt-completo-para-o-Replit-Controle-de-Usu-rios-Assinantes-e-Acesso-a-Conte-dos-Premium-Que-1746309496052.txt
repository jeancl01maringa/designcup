📦 Prompt completo para o Replit — Controle de Usuários, Assinantes e Acesso a Conteúdos Premium
Queremos implementar um painel de gerenciamento de usuários e assinantes dentro da plataforma administrativa. A estrutura e regras devem funcionar da seguinte forma:

🧑‍💻 1. Tipos de usuário:
Usuário Free: é o usuário padrão que se registra diretamente na plataforma (via login/cadastro).

Usuário Assinante (Premium): é um usuário que realizou a compra de algum plano (via Hotmart, Kiwify, etc).

🔁 2. Como o Supabase vai atuar:
O Supabase cria automaticamente um novo usuário ao receber o Webhook de compra aprovada.

Ele vincula esse usuário ao plano correto (baseado no código do plano vindo do Hotmart, por exemplo).

O campo tipo do usuário será alterado para "premium".

Se o usuário cancelar a assinatura, o Supabase deve atualizar o campo tipo para "free" novamente.

🗃️ 3. Estrutura sugerida no Supabase (tabela users):
ts
Copiar
Editar
{
  id: uuid,
  nome: string,
  email: string,
  tipo: 'free' | 'premium',
  plano_id: string,
  data_vencimento: date,
  active: boolean,
  created_at: timestamp
}
📂 4. Conteúdos e Licenças:
Cada arte/postagem deve possuir um campo licenca com valores:

"gratuita" — disponível para todos os usuários

"premium" — somente para usuários com tipo = premium e assinatura ativa

🔐 5. Lógica de acesso ao conteúdo:
Situação do usuário	tipo	Licença da arte	Acesso permitido?
Cadastro gratuito	"free"	"gratuita"	✅ Sim
Cadastro gratuito	"free"	"premium"	❌ Não
Assinante ativo	"premium"	"premium"	✅ Sim
Assinante vencido	"premium"	"premium"	❌ Não (verificar active = false ou data_vencimento < hoje)

🧭 6. Painel de gerenciamento de usuários (tela do Admin):
Criar um menu com dropdown chamado Gerenciamento com duas opções:

Usuários: todos os cadastrados, incluindo gratuitos

Assinantes: somente usuários com tipo = premium e assinatura ativa

O painel deve mostrar:

Nome

Email

Tipo (free ou premium)

Plano vinculado (caso tenha)

Status da assinatura (ativo/inativo)

Vencimento

Botões de editar ou excluir, se necessário

🧩 7. Regras e gatilhos automáticos:
Quando um usuário se cadastra normalmente, ele entra como tipo = free.

Quando um usuário compra um plano via Hotmart/Kiwify, o Supabase atualiza:

tipo = premium

plano_id = id do plano (que deve já estar cadastrado no painel)

data_vencimento = vencimento do plano

active = true

Se o pagamento for cancelado ou expirar, o Supabase deve atualizar:

tipo = free

plano_id = null

active = false

💡 Observações adicionais:
A lógica de acesso (frontend) deve validar o tipo do usuário e status da assinatura antes de liberar o conteúdo.

O botão de acesso (ex: “Editar no Canva”) só aparece se o usuário tiver acesso permitido.

Para usuários free, deve aparecer o botão “Assine para desbloquear” nos conteúdos premium.
